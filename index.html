Bridge the Fundamentals

Your Google cert covers SOC/blue-team basics. Fill the offensive prerequisites: how the web actually works under the hood + comfort in a Linux shell.

⏱ 2–4 weeks · part-time
Master HTTP, DNS, cookies & the request/response cycle core

Everything in web hacking is manipulating a request. Read MDN on HTTP, then NahamSec's "getting started" notes on networking, DNS records, and why HTML structure matters for input/output.

MDN HTTP Guide NahamSec: Getting Started
Get fluent in Linux & the command line free

You'll live in a terminal. Work through Linux Journey, then play OverTheWire's Bandit wargame — it teaches shell skills as a hacking puzzle (SSH, pipes, grep, find).

Linux Journey OverTheWire: Bandit
Read enough JavaScript & basic Python to be dangerous core

You don't need to be a dev. You need to read client-side JS (to spot logic & hidden endpoints) and write small Python scripts to automate. Skim, don't grind.

The Modern JS Tutorial Automate the Boring Stuff (Python)
▶ NahamSec — The No-BS Bug Bounty & Web Hacking Roadmap (15 min — watch this first)
01
Learn the Web Vulnerabilities

The heart of the path. PortSwigger's free Web Security Academy is the gold-standard — 190+ interactive labs maintained by the team that builds Burp Suite. This is where most of your hours go.

⏱ 2–4 months · the main grind
🎯 Strategy: Don't do labs randomly. Follow PortSwigger's structured Learning Paths in order. If a lab stumps you for 30+ min, read the solution, understand it, move on — come back later.
PortSwigger: Server-side topics (SQLi, auth, path traversal, command injection, SSRF, file upload) free core

Start the server-side path. SQL injection and access-control bugs are among the most-reported, highest-paying classes for beginners.

WSA Learning Paths All Topics Index
PortSwigger: Client-side topics (XSS, CSRF, CORS, clickjacking, DOM) free core

Cross-site scripting is the single most common bug bounty finding. Grind every XSS lab until reflected/stored/DOM variants feel automatic.

XSS Labs CSRF Labs
PortSwigger: Advanced topics (access control / IDOR, business logic, request smuggling) free

IDOR and broken access control are beginner-friendly to find and consistently pay out. Business-logic flaws need no fancy payloads — just creative thinking.

Access Control / IDOR Business Logic Flaws
Read the OWASP Top 10 & keep WSTG as a reference free

Frame everything you learn against OWASP's vulnerability taxonomy. The Web Security Testing Guide (WSTG) is your methodical checklist for "what do I test on this page?"

OWASP Top 10 OWASP WSTG
Read "The Web Application Hacker's Handbook" (or Bug Bounty Bootcamp) paid

WAHH is the classic deep reference (written by PortSwigger's founders). "Bug Bounty Bootcamp" by Vickie Li is the more modern, bounty-focused alternative if you want one book.

Bug Bounty Bootcamp (No Starch)
02
Build Your Toolkit

Tools don't find bugs — you do. But you must be fluent with the proxy that sits between your browser and the target. Everything routes through Burp (or Caido).

⏱ runs parallel to Phase 01
Set up & learn Burp Suite (Community is free) core

Intercept, Repeater, Intruder, and the Proxy history are your bread and butter. 87% of HackerOne hunters call it the tool that helps most. Caido is a lighter modern alternative worth trying.

Burp Getting Started Caido (alternative)
Set up a Kali / hacking VM with the core CLI tools free

Install the ProjectDiscovery suite (subfinder, httpx, nuclei, katana, naabu), plus ffuf, and SecLists wordlists. These power your recon in Phase 03.

Get Kali Linux ProjectDiscovery Tools SecLists
Learn ffuf & Nuclei properly free

ffuf for content/parameter fuzzing once you have a hypothesis; Nuclei for fast template-based scanning of known issues. Use them sparingly and legally within scope.

ffuf Nuclei Docs
💡 Tip: Resist "tool collector" syndrome. A hunter who deeply knows Burp + ffuf + the ProjectDiscovery suite beats one with 50 half-understood scripts.
03
Recon & Methodology

Labs hand you the vulnerable page. Real targets don't. Recon is the skill of mapping a target's attack surface — finding the subdomain, endpoint, or forgotten app nobody else looked at.

⏱ 3–6 weeks · ongoing skill
▶ NahamSec — Free Recon Course & Methodology for Bug Bounty Hunters (2025)
Learn subdomain enumeration & endpoint discovery core

subfinder/amass for subdomains → httpx to probe live hosts → katana/waybackurls for endpoints → ffuf for hidden content. Understand why each step matters, not just the commands.

NahamSec Recon Notes OWASP Amass
Study Jhaddix's "The Bug Hunter's Methodology" free core

The canonical talk on building a repeatable end-to-end hunting process. Watch it, then write your own methodology checklist in a notes doc.

Beginner Resources (NahamSec)
Learn GitHub dorking, Google dorking & Shodan free

Exposed API keys, credentials, and config files in public repos are easy, real findings. Shodan reveals exposed services and infrastructure.

Google Hacking DB Shodan
04
Deliberate Practice on Real-ish Targets

Bridge the gap between guided labs and live programs. These platforms give you full, intentionally-vulnerable machines & apps — the closest thing to a real target with a safety net.

⏱ 1–3 months · build the reps
Grind PentesterLab / Hack The Box web tracks paid

HTB's "Bug Bounty Hunter" job-role path and PentesterLab's web badges give structured, escalating challenges. Great for turning lab knowledge into instinct.

Hack The Box PentesterLab
Do free rooms on TryHackMe & HackingHub free

TryHackMe's web fundamentals & OWASP rooms are beginner-gentle. HackingHub (NahamSec-backed) offers free bug-bounty-style challenges.

TryHackMe HackingHub
Start a hunting journal & write practice reports core

For every bug you find in practice, write a real vulnerability report: title, severity, steps to reproduce, impact, remediation. Report-writing is a graded skill on every platform.

HackerOne Report Guide
🏁 Readiness check: Can you take a fresh THM/HTB web box, recon it, find 2–3 vuln classes unaided, and write a clean report? If yes — you're ready for live programs.
05
Go Live — Your First Real Bounties

The leap. Pick programs wisely, stay strictly in scope, manage expectations (your first valid bug may take weeks of duplicates and N/As — that's normal), and iterate.

⏱ ongoing · the real game
Register on HackerOne, Bugcrowd & Intigriti core

Create accounts and read each platform's hacker docs & code of conduct. Intigriti and HackerOne are commonly recommended starting points for newcomers.

HackerOne Bugcrowd Intigriti
Pick ONE program with broad web scope & read the full policy core

Don't scatter. Choose a program with a wide scope (lots of subdomains = more surface), read the scope doc carefully, then spend a session just mapping — no exploitation, pure observation.

HackerOne Program Directory
Try VDPs first to build confidence free

Vulnerability Disclosure Programs don't pay cash but have less competition and lower stakes — perfect for landing your first few accepted reports and building reputation/signal.

VDP Programs
Submit your first report — then keep going core

Expect duplicates and informatives early. Each one teaches you. Read public disclosed reports on HackerOne Hacktivity to learn what real winning bugs look like.

HackerOne Hacktivity (disclosed reports)
⚖️ Rules of engagement: Only test assets explicitly in scope. Never test things you don't have permission to. Unauthorized testing is illegal — the program scope is your legal authorization.
🔁 This is a loop, not a finish line. Real hunters revisit PortSwigger for new research, follow disclosed reports, and refine their recon constantly. Knock out a phase, then circle back deeper. Pace it around your coursework — consistency beats intensity.
